# 05. 실제로 어디에 쓰이는가

## 한 줄 요약
지식 증류는 정확도와 비용 사이의 균형이 필요한 거의 모든 실전 배포 환경에서 쓰입니다.

## 쉬운 비유
대형 트럭이 짐을 가장 많이 싣는다고 해서 모든 길에 트럭만 다닐 수는 없습니다. 골목길에는 작은 차량이 더 적합합니다. AI도 비슷합니다. 가장 큰 모델이 항상 실제 서비스에 맞는 것은 아니기 때문에, 큰 모델의 능력을 더 작은 모델에 옮기는 작업이 필요합니다.

## 핵심 설명
실무에서 지식 증류가 많이 쓰이는 가장 큰 이유는 배포 제약입니다. 서버 비용, 지연 시간, 메모리 사용량, 전력 소비 같은 현실적 제약은 연구용 최고 성능 모델을 그대로 쓰기 어렵게 만듭니다. 지식 증류는 이런 제약 속에서도 가능한 한 높은 성능을 유지하려는 도구입니다.

대표적인 활용 맥락은 아래와 같습니다.

| 상황 | 왜 지식 증류가 필요한가 | 예시 |
| --- | --- | --- |
| 모바일과 엣지 배포 | 메모리와 전력이 제한됨 | 스마트폰 분류 모델, 카메라 분석 모델 |
| 대규모 서버 추론 | 요청 수가 많아 비용이 큼 | 검색 랭킹, 문장 분류, 추천 시스템 |
| 실시간 응답 | 지연 시간이 짧아야 함 | 음성 인식, 실시간 필터링 |
| 모델 제품화 | 연구용 모델이 너무 무거움 | BERT 계열을 DistilBERT나 TinyBERT로 축소 |

컴퓨터 비전에서는 큰 CNN이나 Vision Transformer를 더 작은 student로 압축해, 카메라 장치나 현장 장비에서 돌리기 쉽게 만드는 사례가 많습니다. 음성 분야에서도 큰 음향 모델의 판단을 작은 모델로 옮겨, 실시간 처리가 필요한 환경에 맞추는 경우가 있습니다.

NLP에서는 지식 증류가 특히 널리 알려졌습니다. DistilBERT는 BERT보다 더 작고 빠르면서도 많은 언어 이해 능력을 유지하는 방향을 보여 주었습니다. TinyBERT는 Transformer 내부 구조까지 더 세밀하게 옮기려 했고, MiniLM은 self-attention 자체를 압축의 핵심 대상으로 삼았습니다. 이런 모델들은 정확도만을 조금 양보하고 속도와 메모리 효율을 크게 얻고 싶을 때 실용적입니다.

또 하나 자주 놓치는 점은, 지식 증류가 반드시 온디바이스만을 위한 기술은 아니라는 것입니다. 서버 환경에서도 추론 비용이 매우 중요합니다. 하루에 수백만 건의 요청을 처리하는 서비스에서는 모델이 조금만 가벼워져도 비용 차이가 매우 크게 납니다. 그래서 지식 증류는 클라우드 서비스 운영 관점에서도 중요한 최적화 기법입니다.

## 심화 박스
산업계에서 DistilBERT, TinyBERT, MiniLM이 널리 언급되는 이유는 단순히 작아서가 아닙니다. 이 모델들은 학술적으로도 지식 증류가 실제 제품 수준의 trade-off를 만들 수 있다는 점을 보여 준 상징적인 사례입니다. 즉 지식 증류는 논문 속 테크닉이 아니라, 연구 모델을 서비스 모델로 옮길 때 자주 등장하는 중간 다리입니다.

LLM 시대에도 이 논리는 그대로 이어집니다. 거대한 teacher가 만든 응답 품질을 더 작은 모델이 최대한 흉내 내게 만들면, 배포 가능한 제품형 모델을 더 쉽게 만들 수 있기 때문입니다.

## 자주 생기는 오해
- 지식 증류는 모바일에서만 필요한 기술이 아닙니다. 서버 비용 절감에도 매우 중요합니다.
- 작은 모델을 만든다고 해서 항상 정확도가 크게 떨어지는 것은 아닙니다. 잘 설계된 student는 예상보다 높은 성능을 유지할 수 있습니다.
- 지식 증류는 단순히 속도를 위한 꼼수가 아닙니다. 실제 제품화에서 성능과 비용의 균형을 설계하는 방법입니다.
